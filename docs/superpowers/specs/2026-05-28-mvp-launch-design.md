# DarkTrade MVP 发布设计文档

> 日期：2026-05-28
> 目标：将 DarkTrade 从本地 Demo 升级为可发布的 A 股模拟交易平台
> 定位：面向国内投资新手的游戏化 A 股模拟交易工具

---

## 一、产品定位 & 设计决策

**核心用户**：想体验股票交易但资金有限或不敢真金白银入场的国内投资新手。

**关键决策**：

| 维度 | 决定 |
|------|------|
| 体验风格 | 休闲养成 + 趣味视觉反馈 + 成就勋章 |
| 目标市场 | A 股为主（免费），加密货币及其他市场为会员解锁内容 |
| 社交体系 | 战绩分享图 + 排行榜（周/月/总）+ 好友对战房间 |
| 账号方式 | Supabase 注册/登录 + 游客模式 + 数据迁移 |
| 教程策略 | 可配置完整教程（7 章），不强制；交易时角落随机 Tips |
| 合规态度 | 认真做：免责声明 + 风险提示弹窗 + 行情延迟标注 + 用户协议 |
| 部署方式 | Vercel/CloudFlare Pages 免费部署 → 后续买域名 → PWA |
| 主题视觉 | 亮色暖调（#FFFBF5 暖白底 + #D4A853 琥珀金强调） |
| 名称 | 暂保留 DarkTrade |
| 后续平台 | Web 稳定后独立开发微信小程序（方案 B：独立技术栈） |


**数据流规则：**
- 游客模式 → 所有数据读写走 `HiveLocalRepo`
- 登录模式 → 读写走 `SupabaseRemoteRepo`，本地 `HiveLocalRepo` 保留离线缓存
- 行情数据 → 始终从东方财富 API 获取，不走 Repo 层（不持久化）
- 游客升级登录 → 弹窗确认后，`HiveLocalRepo` → `SupabaseRemoteRepo` 合并迁移

## 四、账号系统 & 游客流程

### 4.1 首次启动流程

1. App 启动 → 检查本地是否有生涯数据
   - 无 → 显示欢迎弹窗，引导创建首个生涯
   - 有 → 跳过
2. 显示免责声明弹窗（仅首次，需点击确认）
3. 进入主界面（游客模式），顶部 `GuestBanner` 持续显示

### 4.2 注册/登录入口

游客模式有两处入口：
- 顶部 `GuestBanner`：「游客模式，数据仅保存在本设备 · 注册 / 登录」
- 个人中心 → 点击「注册 / 登录」按钮

**注册**：用户名（2-20 位字母数字汉字）+ 密码（6 位以上）
**登录**：用户名 + 密码
**实现**：Supabase Auth，username → 映射到 email 字段（拼接 `@darktrade.internal`）

### 4.3 数据迁移（游客 → 登录）

```
登录成功
  → 检查本地是否有生涯数据
    → 有: 弹窗 "检测到本地有 N 个交易生涯，是否导入到你的账户？"
       → 导入: 合并到 Supabase, 本地数据标记为已迁移
       → 不导入: 仅使用线上数据, 本地数据保留
    → 无: 直接进入
```

### 4.4 登录后变化

- `GuestBanner` 消失
- 个人中心显示用户名 + UID
- 所有生涯/持仓/交易记录 → Supabase 远程读写
- 支持退出登录 → 回到游客模式

## 五、新手引导

### 5.1 引导流程

触发条件：本地无生涯数据（即首次启动）。

**步骤 1：欢迎弹窗**
- 欢迎语 + 简介（"免费练习 A 股交易的虚拟平台"）
- 输入生涯名称（默认"我的生涯 #1"）
- 输入初始资金（默认 100,000 ¥）
- 「开始交易」按钮确认

**步骤 2：Tab 高亮**
- 底部 Tab "行情" 高亮闪烁
- 上方浮层 "👆 从这里探索 A 股市场"
- 点击任意位置或 3 秒后自动消失

**设计原则**：两步结束，不拖沓；可跳过（弹窗右上角 X）；不阻塞正常操作。

## 九、教程系统

### 9.1 设计原则

- 可配置但不强制——用户主动打开才看
- 分章节递进，每章独立，不设锁定
- 从第一性原则出发：股票从哪来 → 去哪开户 → 怎么下单 → 怎么看懂 → 怎么不亏

### 9.2 章节

| 章节 | 标题 | 内容概要 |
|------|------|----------|
| 第 1 章 | 一张股票是怎么诞生的 | 公司为什么上市、股票本质、一级 vs 二级市场 |
| 第 2 章 | 我该怎么买到股票 | 券商开户全流程、银证转账、交易软件下载 |
| 第 3 章 | 我的第一次下单 | 买入卖出操作、股票代码、限价单 vs 市价单、一手=100股 |
| 第 4 章 | 看懂行情卡片 | 股价、涨跌幅、成交量、换手率、振幅图解 |
| 第 5 章 | K 线到底是什么 | 阴阳线、影线、常见形态（用故事解释） |
| 第 6 章 | 交易的规则和成本 | T+1、涨跌停板、印花税、佣金、过户费 |
| 第 7 章 | 亏了怎么办 | 止损逻辑、仓位管理、心理陷阱、模拟 vs 真实 |

### 9.3 入口 & 实现
- 行情页顶部 📖 图标 + 个人中心 → "交易学院"
- 内容配置化为 `List<TutorialChapter>`，阅读进度存 Hive

## 十、交易 Tips & 情绪仪表盘

### 10.1 随机 Tips

- 位置：交易页底部 execute 按钮上方空白区域
- 首次进入展示一条，5 秒后消失
- 之后随机触发（~20% 概率），不重复
- MVP 预置 12 条，涵盖 T+1、涨跌停、手续费、红涨绿跌等

## 十一、排行榜 & 好友对战

### 11.1 排行榜

**三个维度**：周榜 / 月榜 / 总榜（按收益率排名 TOP 100）

规则：
- 未登录只能看榜不能上榜（引导注册）
- 已登录自动上榜，用户名脱敏
- 可设置"隐藏昵称"开关

### 11.2 好友对战

**创建房间**：设置名称、初始资金、持续时间（1/3/7/30天），生成邀请码
**对战**：独立生涯副本，不影响主生涯，最大 10 人
**结算**：到期自动排名
