# DarkTrade V2 — 设计文档

> 日期：2026-05-28
> 目标：将 DarkTrade 从本地 Demo 升级为可发布的模拟交易平台
> 定位：面向投资新手的跨市场（加密货币+美股+A股）模拟交易工具

---

## 一、产品定位 & 目标用户

**核心用户**：想体验股票/加密货币交易，但资金有限或不敢真金白银入场的投资新手。

**产品目标**：
- 打开网页即可开始模拟交易，零门槛
- 覆盖加密货币、美股、A 股三大市场
- 游戏化生涯系统，支持创建多个交易生涯互相对比
- 一键导出战绩图片，方便社交分享传播

---

## 二、整体架构

```
┌─────────────────────────────────────────────────┐
│                  Flutter Web App                 │
├─────────────────────────────────────────────────┤
│  UI Layer                                       │
│  行情 Tab / 资产 Tab / 交易 Tab / 个人 Tab        │
│  + 交易记录页 + 生涯管理弹窗 + 新手引导            │
├─────────────────────────────────────────────────┤
│  State Layer (Provider)                         │
│  LiveMarketService  UsStockService  AShareService│
│  PortfolioService  TradeSelectionService         │
│  + AuthService(新增) + CareerService(新增)        │
│  + TradeHistoryService(新增)                     │
├─────────────────────────────────────────────────┤
│  Repository Layer (新增)                         │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │本地 Hive  │  │ 远程 API │  │ 外部行情 API   │  │
│  │(游客数据) │  │(登录用户) │  │ Binance/EastM │  │
│  └──────────┘  └──────────┘  └───────────────┘  │
└─────────────────────────────────────────────────┘
                      │
          ┌───────────┴───────────┐
          ▼                       ▼
    ┌──────────┐           ┌──────────────┐
    │ Supabase │           │ 外部行情 API  │
    │  • 认证   │           │ Binance      │
    │  • 用户表 │           │ 东方财富      │
    │  • 生涯   │           └──────────────┘
    │  • 持仓   │
    │  • 交易记录│
    └──────────┘
```

**数据流原则**：
- 游客模式：所有交易/持仓/生涯数据仅存本地 Hive，刷新不丢失
- 登录模式：数据实时同步到 Supabase，本地保留副本作为离线缓存
- 行情数据：不受登录状态影响，始终从外部 API 获取

## 三、账号系统 & 登录

### 3.1 游客模式（默认）

- 首次打开自动创建首个生涯（默认"生涯 #1"，初始资金 100,000 USDT）
- 所有功能无限制使用
- 页面顶部半透明提示条：「游客模式，数据仅保存在本设备 · 注册 / 登录」
- 游客可随时注册并导入/合并本地数据

### 3.2 注册 & 登录

- **注册**：用户名（4-20 位字母数字）+ 密码（6 位以上）
- **登录**：用户名 + 密码
- **后端**：Supabase Auth（username 映射为 email 字段）
- 登录入口：个人中心顶部 + 游客提示条链接

### 3.3 数据迁移（游客 → 登录）

- 注册/登录成功后弹窗：「是否将本地数据导入到你的账户？」
- 选择"导入"：游客生涯、持仓、交易记录合并到线上
- 选择"不导入"：从线上数据开始，本地数据保留在设备

### 3.4 登录后的变化

- 顶部提示条消失，个人中心显示真实用户名
- 所有数据远程持久化
- 换设备登录后自动恢复

## 四、生涯模式系统

### 4.1 核心概念

每个"生涯"是一个独立的模拟交易存档：

| 属性 | 说明 |
|------|------|
| 生涯名称 | 用户自定义（默认"生涯 #N"） |
| 初始资金 | 创建时自选 1 ~ 100,000,000 USDT |
| 独立持仓 | 每个生涯有自己的买入/卖出记录 |
| 生涯统计 | 总收益率、盈亏、胜率、交易笔数 |

### 4.2 生涯管理

- **创建**：个人中心 → 新建生涯 → 输入名称 + 初始资金（默认 100,000，可自由输入）
- **切换**：资产页顶部生涯选择器，点击弹出切换底部弹窗
- **删除**：生涯列表长按或左滑删除（至少保留一个）
- **存储**：游客存本地 Hive，登录用户存 Supabase

### 4.3 战绩分享图

- 每个生涯提供"导出分享"按钮
- 利用 `RepaintBoundary` + `dart:ui` 将战绩卡片渲染为 PNG
- 通过 `share_plus` 调起系统分享
- 分享图内容：生涯名称、总收益率、盈亏金额、净值迷你曲线、胜率、交易笔数、最佳交易

## 五、本地持久化

### 5.1 技术选型

使用 **Hive** 作为 Flutter Web 的本地持久化方案。

### 5.2 存储结构

```
Box: 'careers'       — 生涯列表（id, name, initialBalance, createdAt）
Box: 'holdings'      — 持仓（careerId, stockId, symbol, amount, avgCost）
Box: 'tradeHistory'  — 交易记录（careerId, type, symbol, qty, price, pnl, timestamp）
Box: 'auth'          — token, username（用于恢复登录态）
Box: 'prefs'         — 首次启动标记、主题偏好等
```

## 六、交易记录页

### 6.1 页面入口

- 资产页 → "交易记录"按钮
- 个人中心 → "交易记录"菜单

### 6.2 页面设计

- 列表展示当前生涯的所有已完成交易
- 每条：时间、币种、买入/卖出标签（绿/红）、数量、成交价、盈亏
- 筛选：近 7 天 / 近 30 天 / 全部
- 空状态：趣味插图 + emoji + 「还没有交易记录，去行情页看看吧 👀」

### 6.3 后端存储（登录模式）

```
table: trade_records
- id (uuid)
- user_id (uuid)
- career_id (uuid)
- type (buy/sell)
- symbol (text)
- name (text)
- market_type (crypto/usStock/aShare)
- quantity (float8)
- price (float8)
- pnl (float8, nullable — 卖出时计算)
- created_at (timestamptz)
```

## 十、新手引导

### 10.1 触发条件

- 检测本地 career 数据为空时触发

### 10.2 引导流程

1. **欢迎弹窗**：「欢迎来到 DarkTrade 🎉 这里可以免费体验股票和加密货币交易！」
2. **创建生涯**：弹窗输入生涯名称 + 初始资金（默认 100,000），确认后自动进入
3. **轻量引导**：底部导航"行情"标签高亮闪烁 + 文案：「从这里开始探索市场」

### 10.3 设计原则

- 不强制看完所有步骤，用户可随时跳过
- 引导过程中不阻塞正常操作

## 十二、实施优先级

| 优先级 | 模块 | 依赖 |
|--------|------|------|

| P1 | 资产页改进（生涯选择器 + 收益卡片） | 生涯系统 |
| P1 | 交易记录页面 | 交易记录持久化 |
| P1 | 交易页微调（可用余额 + 成功动效） | 无 |
| P2 | 账号系统（Supabase 注册/登录） | 无 |
| P2 | 游客/登录双模式 + 数据迁移 | Hive + 账号系统 |
| P2 | 个人中心改造 | 账号系统 |
| P3 | UI 风格活泼化 | 无 |
| P3 | 新手引导 | 生涯系统 |
